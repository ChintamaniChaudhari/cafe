## 1. Product Vision & Core Principles

**Tagline:** The Modern Operating System for Cafés and Restaurants.

**Core Philosophy:** We are not building a simple "QR Menu website." We are laying the foundational architecture for a **Restaurant Operating System (ROS)**. Every engineering decision ensures the platform can seamlessly scale from a single café to a multi-branch SaaS platform without requiring core database rewrites.

### The 6 Immutable Engineering Guardrails
1. **Mobile-First Customers:** 100% of customer interactions occur via smartphone web browsers. Zero desktop styling assumptions are permitted for the customer flow.
2. **Zero-Friction Ordering:** Customers must *never* be prompted to register, download an app, or verify an OTP. The flow is strictly: Scan $\rightarrow$ Select $\rightarrow$ Order.
3. **Sub-Second KDS Synchronicity:** Kitchen tablet displays must reflect incoming orders in <500ms via Asynchronous WebSockets. Polling HTTP endpoints is strictly banned.
4. **Modular Monolith Architecture:** Domain engines (`auth`, `menu`, `orders`) are physically isolated inside the codebase. Cross-engine database joins are prohibited; engines communicate strictly via public internal Python service methods.
5. **Session Continuity:** Customer table identity is locked to a database-backed `dining_session` via cryptographically signed HTTP-Only cookies.
6. **Client Cart Hydration:** Active shopping carts must persist inside the client's `window.localStorage` to survive accidental browser refreshes.
